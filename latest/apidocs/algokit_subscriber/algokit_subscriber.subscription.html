indexer_pre_filter_in_memory(
subscription: algokit_subscriber.types.subscription.TransactionFilter,
) → collections.abc.Callable[[algokit_subscriber.types.indexer.TransactionResult], bool]¶

Create an in-memory pre-filter function based on the subscription parameters.

This is needed to overcome the problem that when indexer matches on an inner transaction, it doesn’t return that inner transaction, it returns the parent. We need to re-run these filters in-memory to identify the actual transaction(s) that were matched.

Parameters:

subscription – The transaction filter parameters

Returns:

A function that applies the pre-filter to a transaction dictionary
